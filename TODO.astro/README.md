# Astro Migration Plan

## Context

The OIML Terminology Harmonization site is currently built with Vue 3 + vite-ssg.
We are migrating to Astro to simplify the build pipeline and leverage Astro's
native static site generation, content collections, and component islands.

## What stays the same

- **Ruby data pipeline** (`scripts/migrate_from_vocab.rb`, `check_consistency.rb`,
  `export_for_vite.rb`) — produces JSON in `web/src/data/`. No change needed.
- **Composables** (`web/src/composables/*.ts`) — pure TypeScript logic. Port
  directly to Astro frontmatter or client-side islands.
- **CSS** (`web/src/styles/*.css`) — Tailwind v4 + design tokens. Port to
  Astro's `src/styles/`.
- **Data files** (`web/src/data/*.json`) — generated by Ruby pipeline. Move to
  `src/data/`.
- **vitest test suite** (`web/src/composables/*.test.ts`) — 57 tests covering
  all business logic. All must pass after migration.

## What changes

- **Framework**: Vue 3 SFCs → Astro components (`.astro` files)
- **Routing**: vue-router → Astro file-based routing
- **SSG**: vite-ssg → Astro built-in SSG
- **Interactivity**: Vue components → Astro islands (Vue or vanilla JS)
